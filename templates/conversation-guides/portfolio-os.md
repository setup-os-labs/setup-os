# Portfolio OS Conversation Guide

Use this when planning a local portfolio assistant.

## Goal

What outcome should the portfolio system support?

## Current Portfolio

What holdings, accounts, exports, or screenshots can it read?

## Investing Philosophy

What rules, risk tolerance, time horizon, target allocation, and constraints should it remember?

## Action Model

Choose one:

- advisory only
- approval required before execution
- limited autonomy for pre-approved rules
- full autonomy

## Alerts

What should trigger alerts, and where should they be sent?

## Safety

What must never happen without explicit approval?

## Existing Tools To Check

Ask whether existing products, open-source finance agents, portfolio tools, or broker integrations can be composed before custom code is built.
