# Vehicle OS Conversation Guide

Use this when planning a local vehicle maintenance and ownership assistant.

## Goal

What vehicle outcomes should the system manage?

## Vehicle Data

List vehicles, service history, receipts, warranties, insurance, registrations, and recurring costs.

## Workflows

What should it track: maintenance, repairs, fuel, tires, insurance, resale value, recalls, or trip readiness?

## Alerts

What should trigger reminders or warnings?

## Action Model

Should it only advise, draft messages, book appointments after approval, or perform recurring actions?

## Existing Tools To Check

Ask whether existing maintenance trackers, calendars, recall APIs, or document stores can be composed.
